# Тест-план

## Цель

Проверить основные пользовательские и API-сценарии мессенджера перед релизом.

## Объект тестирования

- авторизация
- список и поиск чатов
- история сообщений
- отправка сообщений
- редактирование и удаление сообщений
- realtime-обновления
- REST API
- mobile viewport

## Виды тестирования

- функциональное
- smoke
- регрессионное
- интеграционное
- API
- негативное
- адаптивное
- исследовательское

## Критические риски

- невозможность авторизоваться
- потеря отправленного сообщения
- дублирование сообщений
- рассинхронизация UI и API
- редактирование или удаление чужого сообщения
- отсутствие входящих сообщений без перезагрузки
- потеря истории после переключения чата

## Критерии начала

- API и frontend доступны
- тестовые данные загружены
- test API для сброса состояния работает

## Критерии завершения

- smoke-набор прошёл
- нет blocker и critical дефектов
- pass rate регресса не ниже 95 процентов
- все найденные дефекты заведены и приоритизированы
