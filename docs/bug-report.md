# Пример баг-репорта

## Заголовок

[Web][Messages][Critical] После повторного подключения одно отправленное сообщение отображается дважды

## Окружение

- Chrome 149
- Windows 11
- Build 1.0.0
- API localhost:3001
- Web localhost:5173

## Предусловия

Пользователь авторизован. Открыт чат Anna Petrova.

## Шаги

1. Отключить сеть
2. Ввести сообщение
3. Включить сеть
4. Нажать Отправить
5. Дождаться восстановления Socket.IO соединения

## Фактический результат

Одно сообщение появляется в истории дважды. Оба элемента имеют одинаковый текст, но разные DOM-состояния.

## Ожидаемый результат

Сообщение отображается один раз. Повторное событие Socket.IO не создаёт дубль.

## Частота

4 из 5 попыток
