# Стратегия автоматизации

## UI слой

Cypress проверяет критические пользовательские сценарии

- вход
- поиск чата
- отправка
- редактирование
- удаление
- входящее realtime-сообщение
- mobile viewport

## API слой

API-тесты проверяют

- авторизацию
- коды ответа
- структуру ответа
- обязательность текста
- авторизацию через Bearer token
- создание сообщения

## Стабильность

- состояние сбрасывается через POST /api/test/reset
- тесты не зависят друг от друга
- используются data-testid
- секретные данные не выводятся в Cypress log
- realtime-тест использует test API вместо второго браузера
- retry включён только в run mode
